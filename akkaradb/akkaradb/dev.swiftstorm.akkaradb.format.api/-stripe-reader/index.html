StripeReader

interface StripeReader : Closeable

Reader abstraction for sequentially consuming stripes from a segment.

A stripe is the unit of redundancy in AkkaraDB's append-only format: it consists of multiple fixed-size lane blocks (k data lanes and optionally m parity lanes). Each stripe may contain one or more encoded records.

Implementations are responsible for:

  • Reading the full set of lane blocks that make up a stripe.

  • Validating and optionally reconstructing missing lanes using a ParityCoder.

  • Exposing decoded payload slices and retaining ownership of the underlying lane buffers until released.

Concurrency: unless stated otherwise by the implementation, calls are single-consumer.
